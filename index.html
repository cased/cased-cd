Quick Start

Install with Helm (Recommended)

# Add the Cased Helm repository
helm repo add cased https://cased.github.io/cased-cd
helm repo update

# Install Cased CD in the argocd namespace
helm install cased-cd cased/cased-cd \
  --namespace argocd \
  --create-namespace

Access Cased CD at http://localhost:8080 (via port-forward) or configure an Ingress for external access.

Install with kubectl

Note: This installs the standard (non-enterprise) version only. For enterprise features, see the Enterprise tab.

# Apply the manifests
kubectl apply -f https://cased.github.io/cased-cd/install.yaml

# Access via port-forward
kubectl port-forward -n argocd svc/cased-cd 8080:80

Important: The static install.yaml assumes ArgoCD is installed in the argocd namespace with the default service name argocd-server. If your setup is different, you'll need to either:

  1. Use Helm instead (recommended - allows easy configuration with --set argocd.server=...)
  2. Edit the install.yaml before applying - download it, find line 164, and update the ARGOCD_SERVER environment variable to match your ArgoCD server location

Run with Docker

docker run -d \
  -p 8080:80 \
  -e ARGOCD_SERVER=http://argocd-server.argocd.svc.cluster.local:80 \
  ghcr.io/cased/cased-cd:latest

Prerequisites: Existing ArgoCD installation (v2.0+), Kubernetes cluster with default StorageClass for audit log PVC, and enterprise credentials for accessing the private container image

Authentication

The enterprise backend image (registry.cased.com/cased/cased-cd-enterprise) requires authentication. Enterprise customers should contact Cased support to receive their registry token, then create an imagePullSecret:

# Contact support@cased.com to receive your registry token
# Then create the secret in your cluster:
kubectl create secret docker-registry cased-cd-registry \
  --docker-server=registry.cased.com \
  --docker-username=token \
  --docker-password="YOUR_REGISTRY_TOKEN" \
  --namespace=argocd

Benefits: Cased's registry proxy provides instant token revocation, scoped access, and complete audit trails of all image pulls.

Storage Requirements

Enterprise includes a persistent audit trail (enabled by default) that requires storage. The chart automatically uses your cluster's default storage class.

Note: If your cluster (e.g., Talos, vanilla Kubernetes, custom clusters) doesn't have a default storage class, you can either:

  • Specify a storage class explicitly:
    --set enterprise.persistence.storageClass=<your-storage-class>
  • Or disable persistent audit logging (audit events logged to stdout only):
    --set enterprise.auditTrail.enabled=false

Common storage classes by platform:

  • AWS EKS: gp2, gp3
  • Google GKE: standard, standard-rwo
  • Azure AKS: default, managed-premium
  • k3d/k3s: local-path

Deploy with Helm

# Add the Cased Helm repository
helm repo add cased https://cased.github.io/cased-cd
helm repo update

# Install Cased CD Enterprise
helm install cased-cd cased/cased-cd \
  --namespace argocd \
  --set enterprise.enabled=true \
  --set enterprise.persistence.size=10Gi \
  --set imagePullSecrets[0].name=cased-cd-registry

Deploy with ArgoCD

If you're using ArgoCD to manage your applications, add imagePullSecrets to your Application's Helm values:

apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: cased-cd
  namespace: argocd
spec:
  project: default
  source:
    repoURL: https://cased.github.io/cased-cd
    chart: cased-cd
    targetRevision: 0.1.20
    helm:
      values: |
        # Registry secret for enterprise image
        imagePullSecrets:
          - name: cased-cd-registry

        # Enterprise configuration
        enterprise:
          enabled: true
          auditTrail:
            enabled: true
          image:
            repository: registry.cased.com/cased/cased-cd-enterprise

        # Your ArgoCD server
        argocd:
          server: "https://argocd-server.argocd.svc.cluster.local"

  destination:
    server: https://kubernetes.default.svc
    namespace: argocd

  syncPolicy:
    automated:
      prune: true
      selfHeal: true

Important: Create the cased-cd-registry secret before syncing the Application. ArgoCD cannot use --set flags, so you must configure imagePullSecrets in the helm.values section.

Deploy with kubectl

Option 1: Full Installation (with persistent audit trail)

# First, create the imagePullSecret for the private enterprise image
kubectl create secret docker-registry cased-cd-registry \
  --docker-server=registry.cased.com \
  --docker-username=token \
  --docker-password="YOUR_REGISTRY_TOKEN" \
  --namespace=argocd

# Apply the enterprise manifests
kubectl apply -f https://cased.github.io/cased-cd/install-enterprise.yaml

Note: This manifest includes a PVC for audit logs. Requires a cluster with a default storage class.

Option 2: Minimal Installation (no storage required)

For clusters without a default storage class (Talos, bare-metal, etc.), use the minimal manifest:

# First, create the imagePullSecret
kubectl create secret docker-registry cased-cd-registry \
  --docker-server=registry.cased.com \
  --docker-username=token \
  --docker-password="YOUR_REGISTRY_TOKEN" \
  --namespace=argocd

# Apply the minimal enterprise manifests (no PVC)
kubectl apply -f https://cased.github.io/cased-cd/install-enterprise-minimal.yaml

Note: The minimal manifest disables persistent audit logging. Audit events are still logged to stdout and can be collected via your log aggregation system.

Configure External Access (Ingress)

helm upgrade cased-cd cased/cased-cd \
  --namespace argocd \
  --set enterprise.enabled=true \
  --set imagePullSecrets[0].name=cased-cd-registry \
  --set ingress.enabled=true \
  --set ingress.className=nginx \
  --set ingress.hosts[0].host=cased-cd.example.com \
  --set ingress.hosts[0].paths[0].path=/ \
  --set ingress.hosts[0].paths[0].pathType=Prefix

Customize Audit Storage

# Use larger PVC (default: 10Gi supports ~20M events)
helm install cased-cd cased/cased-cd \
  --namespace argocd \
  --set enterprise.enabled=true \
  --set enterprise.persistence.size=50Gi \
  --set enterprise.persistence.storageClass=fast-ssd \
  --set imagePullSecrets[0].name=cased-cd-registry

Verify Deployment

# Check pods are running
kubectl get pods -n argocd -l app.kubernetes.io/name=cased-cd

# Check audit PVC was created
kubectl get pvc -n argocd -l app.kubernetes.io/component=enterprise

# View audit logs
kubectl exec -n argocd deployment/cased-cd-enterprise -- \
  tail -f /data/audit/events.jsonl

Configuration

Connecting to ArgoCD

Cased CD needs to know where your ArgoCD server is located. The default configuration assumes ArgoCD is installed in the argocd namespace with the service name argocd-server.

Default value: http://argocd-server.argocd.svc.cluster.local:80

If your ArgoCD installation uses a different namespace or service name, you must configure the ARGOCD_SERVER environment variable:

Helm (recommended):

# values.yaml
argocd:
  server: "http://argocd-server.argocd.svc.cluster.local:80"

# Examples for different setups:
# - Different namespace: "http://argocd-server.my-namespace.svc.cluster.local:80"
# - Different service name: "http://my-argocd.argocd.svc.cluster.local:80"
# - External ArgoCD: "https://argocd.example.com"

kubectl (requires manual edit):

# Download the install.yaml
curl -o install.yaml https://cased.github.io/cased-cd/install.yaml

# Edit line 164 to match your ArgoCD server location
# Then apply
kubectl apply -f install.yaml

Docker:

docker run -e ARGOCD_SERVER=https://argocd.example.com ghcr.io/cased/cased-cd:latest

Authentication

Cased CD uses the same authentication as ArgoCD. Log in with your ArgoCD credentials:

# Get the admin password
kubectl -n argocd get secret argocd-initial-admin-secret \
  -o jsonpath="{.data.password}" | base64 -d
  • Username: admin
  • Password: (from command above)

CLI Tool

The cased-cd CLI helps you debug installations and understand your Kubernetes environment.

Installation

Download pre-built binaries from the GitHub releases page for macOS, Linux (amd64/arm64), or Windows.

Example (macOS):

curl -LO https://github.com/cased/cased-cd/releases/latest/download/cased-cd-darwin-arm64.tar.gz
tar xzf cased-cd-darwin-arm64.tar.gz
sudo mv cased-cd-darwin-arm64 /usr/local/bin/cased-cd

Commands

  • cased-cd context - Show current Kubernetes context and cluster info
  • cased-cd doctor - Health check for your Cased CD installation
  • cased-cd access - Shows how to access your Cased CD installation
  • cased-cd version - Display component versions
